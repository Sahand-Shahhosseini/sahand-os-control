\section{Purpose}
Define a minimal, auditable documentation protocol for the account:
\begin{itemize}
\item 7 core docs (execution, evidence, governance)
\item 14 extended docs (domain and project packaging)
\end{itemize}

\section{Hard Rules}
\begin{enumerate}
\item Executor-mode: user is click-only.
\item Fail-closed: no silent success.
\item Evidence pack required for meaningful work.
\item No placeholders in released or governance documents.
\end{enumerate}

\section{Core 7 Docs (outline)}
\begin{enumerate}
\item DOC01 — Scope and Objective
\item DOC02 — Inputs and Outputs
\item DOC03 — Acceptance Gates
\item DOC04 — Reproducible Evidence Chain
\item DOC05 — Storage and Archive Policy
\item DOC06 — Review Packet Protocol
\item DOC07 — Release and Lock Policy
\end{enumerate}

\section{Extended 14 Docs (outline)}
The extended set refines domain-specific deliverables and reviewer bundles. Each extended doc must:
\begin{itemize}
\item state scope and claims,
\item define measurable acceptance,
\item map to tests or proof obligations,
\item declare storage location and hashes.
\end{itemize}

\section{Versioning}
Use the Scope Ladder: v0 $\rightarrow$ RC0 $\rightarrow$ RC1 $\rightarrow$ APPROVED\_EXT $\rightarrow$ LOCKED\_v0.1.
